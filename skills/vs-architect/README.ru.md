# VS Architect - distribution-level промптинг для LLM

🇬🇧 [English version](README.md)

Скилл для [opencode](https://github.com/opencode-ai/opencode): **Verbalized Sampling** (arXiv 2510.01171), чтобы модель не схлопывалась в один и тот же ответ.

## Verbalized Sampling

Просишь LLM «сделай X» - получаешь самый типичный ответ (mode prototype). Это **mode collapse**: модель снова и снова выдаёт один шаблон, даже когда хороших вариантов много.

**Verbalized Sampling (VS)** меняет форму промпта. Вместо «сделай X» - «дай 5 вариантов X с вероятностями успеха». Модель перестаёт залипать на одном прототипе и выдаёт распределение ближе к реальному разнообразию решений.

Статья: arXiv 2510.01171v3 (ICLR 2026) -
*Verbalized Sampling: How to Mitigate Mode Collapse and Unlock LLM Diversity.*

## Какие задачи закрывает

| Проблема | Что делает VS |
|----------|---------------|
| LLM всегда одно и то же | Distribution-level промпт раскрывает разброс |
| Неясно, какой подход взять | Варианты с вероятностями - видно, что перспективнее |
| Диагностика без одной гипотезы | VS-CoT: дерево гипотез с проверками |
| Выбор архитектуры | VS-Standard: сравнение подходов, плюсы и минусы |
| Нужно много идей или данных | VS-Multi: несколько раундов, 10+ вариантов |

## На выходе

- От 3 до 10+ вариантов с оценкой шанса успеха
- Понятная классификация задачи
- Готовый execution prompt для выбранного варианта
- Меньше кругов «а давай ещё раз, вдруг другое»

## Когда звать

- Выбор архитектуры или подхода из нескольких
- Прямой промпт даёт однотипные ответы (mode collapse)
- Диагностика с неизвестной причиной
- Гипотезы, синтетические данные, тест-кейсы
- Креатив, где важно разнообразие

## Когда не звать

- Простой скрипт с очевидным решением - проще написать напрямую
- Факты (2 плюс 2, столица Франции)
- Стабильный продакшн-код - прямой промпт надёжнее

## Установка

```bash
# Клонировать репозиторий со скиллами
git clone git@github.com:dimkurilo/opencode-skills.git ~/Projects/opencode-skills

# Симлинк в конфиг opencode
ln -sf ~/Projects/opencode-skills/skills/vs-architect ~/.config/opencode/skills/vs-architect
```

Или скопировать папку:

```bash
cp -r ~/Projects/opencode-skills/skills/vs-architect ~/.config/opencode/skills/vs-architect
```

## Использование

После установки opencode сам найдёт скилл и предложит его, если задача совпадает с описанием.

## Паттерны

| Паттерн | Когда | Что делает |
|---------|-------|------------|
| **A** VS-Standard | Средняя сложность, 3-5 вариантов | Один вызов, k вариантов с вероятностями |
| **B** Diversity Tuning | Прямой промпт сыпет однотипным | Фильтр по порогу вероятности |
| **C** VS-CoT | Неопределённость, диагностика, стратегия | Сначала рассуждения, потом распределение |
| **D** VS-Multi | Нужно 10+ вариантов | Несколько раундов, каждый исключает прежние идеи |
| **E** External Collapse | Внешние ограничения, о которых LLM не знает | Ты взвешиваешь варианты по своим критериям |
| **F** VS-Refine | Хорошие идеи надо дожать | Два прохода: варианты, потом уточнение лучших |
| **G** VS-Ensemble | Критическое решение, нужна перекрёстная проверка | Два независимых VS-промпта, сравнение результатов |
