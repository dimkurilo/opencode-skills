# VS Architect — Distribution-Level Промптинг для LLM

Скилл для агентов [opencode](https://github.com/opencode-ai/opencode), использующий **Verbalized Sampling** (arXiv 2510.01171) для преодоления mode collapse у LLM.

🇬🇧 [English version](README.md)

## Что такое Verbalized Sampling?

Когда вы просите LLM «сделай X», модель выдаёт самый типичный ответ (mode prototype).
Это называется **mode collapse** — модель каждый раз сходится к одному и тому же шаблонному решению,
даже если есть много хороших альтернатив.

**Verbalized Sampling (VS)** решает эту проблему, меняя структуру промпта.
Вместо `"сделай X"` мы просим: `"дай 5 вариантов решения X с вероятностями успеха"`.
Модель перестаёт схлопываться в один прототип и начинает выдавать распределение вариантов,
приближенное к реальному разнообразию возможных решений.

Основан на статье arXiv 2510.01171v3 (ICLR 2026):
*"Verbalized Sampling: How to Mitigate Mode Collapse and Unlock LLM Diversity."*

## Какие проблемы решает?

| Проблема | Как VS решает |
|----------|--------------|
| LLM всегда даёт одно и то же решение | Distribution-level промпт «вскрывает» разнообразие |
| Не знаешь, какой подход выбрать | VS генерирует варианты с вероятностями — видно, что перспективнее |
| Сложная диагностика без единой гипотезы | VS-CoT паттерн строит дерево гипотез с проверками |
| Сомнения при выборе архитектуры | VS-Standard — сравнение подходов с pros/cons |
| Нужно много идей или данных | VS-Multi — итеративная генерация, 10+ вариантов |

## Что вы получите?

- От 3 до 10+ вариантов решения с оценкой вероятности успеха для каждого
- Чёткую классификацию задачи, понятную пользователю
- Готовый execution prompt для выбранного варианта — бери и делай
- Экономию времени: не нужно переспрашивать LLM 5 раз в надежде на другой ответ

## Когда использовать

- Выбор архитектуры или подхода из нескольких вариантов
- Прямой промпт даёт однотипные ответы (mode collapse)
- Диагностика с неизвестной причиной проблемы
- Генерация гипотез, синтетических данных, тест-кейсов
- Креативные задачи, где нужно разнообразие

## Когда НЕ использовать

- Простые скрипты с очевидным решением (проще написать напрямую)
- Фактические запросы (2+2, столица Франции)
- Стабильный продакшн-код — прямой промпт надёжнее

## Установка

```bash
# Клонировать репозиторий со скиллами
git clone git@github.com:dimkurilo/opencode-skills.git ~/Projects/opencode-skills

# Симлинк в конфиг opencode
ln -sf ~/Projects/opencode-skills/skills/vs-architect ~/.config/opencode/skills/vs-architect
```

Или просто скопировать папку:

```bash
cp -r ~/Projects/opencode-skills/skills/vs-architect ~/.config/opencode/skills/vs-architect
```

## Использование

После установки агент opencode автоматически обнаружит скилл и предложит загрузить его,
когда задача соответствует описанию скилла.

## Паттерны

| Паттерн | Когда | Что делает |
|---------|-------|------------|
| **A** VS-Standard | Средняя сложность, 3–5 вариантов | Один вызов → k вариантов с вероятностями |
| **B** Diversity Tuning | Однотипные ответы от прямого промпта | Фильтрация по порогу вероятности |
| **C** VS-CoT | Неопределённость, диагностика, стратегия | Сначала рассуждения, потом распределение |
| **D** VS-Multi | Нужно 10+ вариантов | Несколько раундов, каждый исключает предыдущие идеи |
| **E** External Collapse | Внешние ограничения, о которых LLM не знает | Пользователь взвешивает варианты по своим критериям |
| **F** VS-Refine | Интересные идеи нуждаются в проработке | Два прохода: варианты → уточнение лучших |
| **G** VS-Ensemble | Критическое решение нуждается в перекрёстной проверке | Два независимых VS промпта, сравнение результатов |
