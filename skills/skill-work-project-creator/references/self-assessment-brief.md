# Self-assessment brief (≥10% roles)

Ask the candidate model/CLI to answer **on disk**:

1. What harness capabilities do I have? (skills, subagents, memory, MCP)
2. What am I weak at relative to this program's domain?
3. What % load is realistic? What must I not own?
4. Evidence from prior runs (paths), not vibes.

Output path: `audits/<lane>/SELF-ASSESSMENT-<date>.md`.

Gate: do not lock role ≥10% without this file or explicit human override in STATUS.
