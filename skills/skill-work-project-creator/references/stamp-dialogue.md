# Stamp Dialogue (first-class)

```
STAMP DIALOGUE
──────────────
Actors:     Author (usually orchestrator), Reviewer (usually executor or auditor)
Session:    Same Reviewer session for rounds 1..N of ONE artifact set
Authority:  STAMP file on disk (AGREED YES|NO + open deltas + content hash)
Transport:  Orca / paste / any — irrelevant to truth
Loop:       Author writes → Reviewer stamps → Author patches ONLY open deltas → re-stamp
Stop:       AGREED=YES | MAX_ROUNDS=5 | stalemate → human
Forbidden:  Chat/subject AGREED if stamp ≠ YES
            Mid-loop checklist expansion except CRITICAL safety
            Cold new session per micro-fix
```

Applies to: program SPEC alignment, wave SPEC/PLAN, skill-spec iteration, any multi-agent disk contract.
