# Harness > model

Roles come from **inventory of CLIs, skills, MCP, subscriptions** — not model marketing prestige.

1. Run `inventory_harness.sh`.
2. Map: who has domain skills? who has agentic runtime (sessions, subagents)? who is cheap for F-04?
3. Self-assessment for ≥10% roles before lock.
4. Degraded 1–2 model modes are valid — document in SPEC.

Anti-pattern: always-on 4-model architecture panel for every project.
