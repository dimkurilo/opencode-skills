# H-panel legitimacy rules

1. Each H is a **dominance claim** with PASS/FAIL criteria (not vibes).
2. Outcomes: `PASS | FAIL | PARTIAL` (PARTIAL must name sub-capability).
3. If ≥3 Hs FAIL/PARTIAL on single-model dominance → emit **`ROLE_ARCHITECTURE_REQUIRED`** (success of discovery).
4. Cap: ≤5 Hs, 1 worker each, ~45–60 min wall; no recursive re-panel.
5. After G0 → archive to `audits/_legacy/`; never session-start load.
