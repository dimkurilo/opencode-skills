# Project Bootstrap — Генератор инфраструктуры по Agent Playbook

🇬🇧 [English version](README.md)

Скилл для агентов [opencode](https://github.com/opencode-ai/opencode), который создаёт полную агентскую инфраструктуру для нового проекта по стандарту [Agent Playbook](https://agents.md) (v0.0.5).

## Что такое Agent Playbook?

Agent Playbook — стандарт организации AI-агентской инфраструктуры в программных проектах. Он задаёт единую структуру директории `.agents/` с модульными правилами, скиллами, командами, агентскими персонами и персистентной памятью — чтобы агенты могли эффективно работать между сессиями, не теряя контекст.

## Что делает этот скилл?

Когда вы описываете новый проект (в любой форме — свободный текст, ссылка на спек, голосовая заметка), этот скилл:

1. **Анализирует** задачу на «тёмные пятна» — ищет в интернете актуальный синтаксис CLI, документацию API и лучшие практики
2. **Проверяет существующие проекты** — находит и переиспользует релевантный контекст из других ваших `.agents/`-настроек
3. **Генерирует инфраструктуру** — создаёт ровно то, что нужно проекту:

| Модуль | Что создаёт |
|--------|-------------|
| `AGENTS.md` | Главный манифест с описанием проекта, деревом архитектуры, таблицей Loaded Context, критическими правилами |
| `.agents/memory/MEMORY.md` | Персистентная память: CONFIRMED_FACTS, инструменты, ограничения, анти-паттерны |
| `.agents/rules/general.md` | Базовые правила: язык, верификация, запрет галлюцинаций, обработка ошибок |
| `.agents/rules/*.md` | Доменные правила (если в задаче есть повторяемые операции) |
| `.agents/agents/*.md` | Персоны сабагентов (если в задаче есть разные роли) |
| `.agents/commands/*.md` | Слеш-команды (если в задаче есть повторяемые команды) |
| `.agents/skills/*/` | Многошаговые workflow с SKILL.md + WORKFLOW.md + скриптами |
| `docs/` | Формальные спецификации (если пользователь передал PRD/спек) |

**Ключевой принцип:** создаёт только то, что задача реально требует. Без раздувания.

## Когда использовать

- Начинаете новый проект и хотите агентскую инфраструктуру с первого дня
- Применяете стандарт Agent Playbook к существующему проекту
- У вас есть свободное описание задачи и вы хотите превратить его в структурированный проект
- Пользователь говорит: «создай структуру», «разверни агента», «настрой проект», «bootstrap», «запусти скилл для проекта»

## Когда НЕ использовать

- Мелкие правки или фиксы внутри уже настроенного проекта
- Аудит существующей инфраструктуры
- Задачи, не связанные с созданием новой структуры проекта

## Что вы получите

- Полное дерево `.agents/` со всеми нужными модулями
- `AGENTS.md` в корне проекта с таблицей Loaded Context
- CLI-синтаксис, API-эндпоинты и лучшие практики, найденные в интернете и сохранённые в память
- Переиспользованный контекст из других ваших проектов (SSH-конфиги, известные грабли, рабочие процедуры)
- Никаких паролей, токенов, секретов — никогда

## Установка

```bash
# Клонировать репозиторий со скиллами
git clone git@github.com:dimkurilo/opencode-skills.git ~/Projects/opencode-skills

# Симлинк в конфиг opencode
ln -sf ~/Projects/opencode-skills/skills/project-bootstrap ~/.config/opencode/skills/project-bootstrap
```

Или просто скопировать папку:

```bash
cp -r ~/Projects/opencode-skills/skills/project-bootstrap ~/.config/opencode/skills/project-bootstrap
```

## Использование

После установки агент opencode автоматически обнаружит скилл и предложит загрузить его, когда вы описываете новый проект или говорите «создай структуру», «разверни агента» и т.д.

## Фреймворк принятия решений

При анализе задачи скилл применяет такую логику:

| Ситуация | Решение | Пример |
|----------|---------|--------|
| Частая/защищённая операция | MCP (упомянуть в AGENTS.md) | Confluence, Jira, GitHub API |
| Редкая/сложная операция | CLI-команда (в commands/) | `gh pr create`, `aws s3 sync` |
| Цепочка CLI+MCP | Скрипт в `skills/*/scripts/` | yt-dlp → метаданные → S3 |
| Повторяемый workflow | Скилл с WORKFLOW.md | «бекап сервера» = 5 шагов |

## Структура репозитория

```
project-bootstrap/
├── SKILL.md                    # Инструкции для агента (загружается opencode)
├── README.md                   # Английская версия
├── README.ru.md                # Этот файл — русское описание
├── references/
│   └── playbook.md             # Сокращённый референс Agent Playbook v0.0.5
└── assets/
    └── templates/
        ├── AGENTS.md.tmpl       # Шаблон главного манифеста
        ├── MEMORY.md.tmpl       # Шаблон персистентной памяти
        ├── general-rule.md.tmpl # Шаблон базовых правил
        ├── rule.md.tmpl         # Шаблон доменного правила
        ├── agent-persona.md.tmpl # Шаблон персоны сабагента
        └── command.md.tmpl      # Шаблон слеш-команды
```
